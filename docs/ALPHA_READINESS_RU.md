## Как добавить анимированный кубик с реальными гранями

1. Источник случайности оставляем только на backend (`POST /game/roll` уже возвращает `rolled`).
2. На frontend не генерируем число, а только анимируем ожидание:
   - до ответа API показываем вращение/перекатывание;
   - после ответа фиксируем грань из `rolled`.
3. Для “реального” визуала используем один из вариантов:
   - 2D sprite-sheet из 6 граней (`dice-sprite.png`);
   - 3D-куб (CSS transform) с шестью `<div>`, где финальный класс зависит от `rolled`.
4. Синхронизация с движением фишки:
   - шаг 1: `roll` -> анимация кубика `400-700ms`;
   - шаг 2: подсветка path клеток `N * 90ms`;
   - шаг 3: открытие CTA `Купить/Пропустить`.

## Минимальный чеклист для alpha-теста

1. Вход/регистрация по email и телефону.
2. Админ создает сессию, активирует и добавляет минимум 8 клеток.
3. Игрок делает бросок в допустимое окно.
4. Покупка на клетке и в секретном магазине.
5. Админ начисляет баллы, игрок видит уведомление при входе.
